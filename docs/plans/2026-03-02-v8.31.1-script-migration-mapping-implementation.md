# Upstream v8.31.1 Script Migration Mapping Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build an execution-ready, fully traceable script migration matrix from upstream `v8.31.1`, copy non-migrated scripts into `go`, and map migrated scripts to concrete Go files/methods.

**Architecture:** Use `docs/architecture/script-differences.md` as the source-of-truth matrix. Freeze upstream baseline first, then process rows by strategy (`COPY_FROM_MAIN` or `MIGRATE_TO_GO`). For migrated scripts, map ownership to domain-appropriate `internal/*` files/methods instead of one-to-one script translation.

**Tech Stack:** Git (`main` + `upstream/v8.31.1`), Markdown docs, Go package layout (`internal/*`), shell tooling (`rg`, `awk`, `sed`, `wc`), `go test`.
