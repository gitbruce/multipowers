# SH -> Go Migration Backlog (P0/P1/P2)

日期：2026-03-05  
来源：`docs/architecture/script-differences.md` 功能级审计结论

## 目标

将 `main` Shell 架构中仍有价值的能力，按优先级迁移到 `go` 分支，优先补齐上线风险项，避免关键功能缺失。

---

## 排期口径

- `P0`：发布阻断/高风险安全或核心能力缺口，必须先做。
- `P1`：强价值能力，建议紧随 P0。
- `P2`：优化与增强，按资源滚动推进。

估算口径（单人）：
- `S` = 0.5~1.5 天
- `M` = 2~4 天
- `L` = 5~8 天

---

## P0（必须优先）

| ID | 任务 | 范围 | 交付物 | 验收标准（DoD） | 依赖 | 估算 |
|---|---|---|---|---|---|---|
| P0-1 | `extract` 能力闭环 | `mp extract` 命令、`internal/extract` 核心包、测试 | `internal/extract/*`、`internal/cli` 路由、`*_test.go`、命令文档 | 1) `mp extract --prompt ... --json` 可用；2) 至少 1 条端到端用例；3) 错误码和输出契约固定；4) `go test ./...` 通过 | 无 | `L` |
| P0-2 | 外部输入安全包装 | URL 校验、内容信任边界、输入净化 | `internal/security` 或 `internal/inputguard`、接入点（extract/research）测试 | 1) 禁止本地网段与元数据地址；2) 非可信内容进入统一包装；3) 安全回归测试覆盖；4) 默认开启保护策略 | P0-1（接入 extract） | `M` |

### P0 实施拆解

1. 命令面：在 `internal/cli` 增加 `extract` 子命令入口与参数校验。  
2. 领域层：新增 `internal/extract` 的请求/响应结构、核心处理器、错误分类。  
3. 安全层：新增 URL 白名单/黑名单与内容 framing 机制，并在入口强制执行。  
4. 测试层：单测 + 集成测试（正常输入、恶意输入、超长输入、超时）。  
5. 文档层：更新 `docs/CLI-REFERENCE.md` 与架构差异文档状态。

## P1（高价值，建议下一迭代）

| ID | 任务 | 范围 | 交付物 | 验收标准（DoD） | 依赖 | 估算 |
|---|---|---|---|---|---|---|
| P1-1 | 成本可见性命令 | 预估与事后报告（基于 benchmark 记录） | `mp cost estimate`、`mp cost report`、聚合逻辑与测试 | 1) 执行前给出成本区间；2) 执行后可按 run/model 汇总；3) 报告可 JSON 输出；4) 单测覆盖关键计算逻辑 | 复用 `internal/benchmark/*` | `M` |
| P1-2 | checkpoint 恢复机制 | 长流程失败恢复、断点续跑 | `internal/checkpoint/*` 或 `internal/orchestration` 扩展、CLI 开关、测试 | 1) 支持保存/加载 checkpoint；2) 异常中断后可恢复；3) 不破坏现有运行模式；4) 集成测试覆盖恢复路径 | P0-1（优先用于 extract/长任务） | `L` |

### P1 实施拆解

1. 成本：定义估算模型（tokens、provider、fallback）和输出契约。  
2. 恢复：定义 checkpoint schema（版本、阶段、步骤、上下文摘要、时间戳）。  
3. 编排：在 `workflow_adapter/executor` 增加可恢复入口。  
4. 回归：失败注入测试（超时、进程中断、部分成功）。

## 里程碑建议

1. 里程碑 A（1~2 周）：完成 `P0-1` + `P0-2`。  
2. 里程碑 B（2~3 周）：完成 `P1-1`，并启动 `P1-2`。  
3. 里程碑 C（3~5 周）：完成 `P1-2`，推进 `P2` 的性能与测试补齐。

---

## 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 抽象过度导致落地慢 | 延误 P0 | 先做最小可用契约，避免一次性大而全设计 |
| 安全策略误杀正常输入 | 影响可用性 | 规则分级 + 可观测日志 + 白名单机制 |
| checkpoint 引入状态一致性问题 | 恢复失败 | schema 版本化 + 原子写 + 恢复前校验 |
| 成本估算偏差争议 | 信任下降 | 输出区间 + 标注“估算值” + 历史校准 |

## 完成定义（全局）

- 代码：核心路径实现完成，接口稳定。  
- 测试：单测/集成测试通过，关键异常路径有覆盖。  
- 文档：CLI 与架构文档同步更新。  
- 证据：在 `docs/plans/evidence/` 留存验证记录（命令输出/测试结果/样例）。
