# Upstream v8.31.1 Script Migration Mapping Design

Date: 2026-03-02  
Status: Approved

## 1. Context

Based on `docs/plans/2026-03-02-no-shell-hybrid-rearchitecture-design.md`, the runtime target is a hybrid architecture:
- Go provides deterministic atomic capabilities.
- Markdown skills provide reasoning/orchestration.

Current `docs/architecture/script-differences.md` now has a full script inventory and coverage classification, but it still needs to be upgraded into an implementation-ready migration matrix that explicitly answers:
1. Which scripts are copied as-is from `main` to `go`.
2. Which scripts are migrated into Go design.
3. For each migrated script, which Go file/method now owns the behavior.

## 2. Goal

Produce an execution-ready migration plan and mapping matrix aligned to upstream `v8.31.1` baseline, with full per-file traceability and domain-consistent Go landing targets.

## 3. Scope

### In Scope

- Freeze `main` baseline to upstream `https://github.com/nyldn/claude-octopus` tag `v8.31.1`.
- Rebuild script matrix from that baseline.
- Mark each script as:
  - `COPY_FROM_MAIN` (no migration)
  - `MIGRATE_TO_GO` (logic migration)
- Fill mapping fields in `docs/architecture/script-differences.md` for all scripts.
- Enforce mapping rule:
  - Same-name test scripts may map uniformly to `internal/*_test.go`.
  - All other scripts must be mapped per-file by domain to concrete Go file + method(s).

### Out of Scope

- Reintroducing shell runtime logic.
- One-to-one mechanical translation of every shell file.
- Immediate implementation of all mapped targets in this design step.

## 4. Architecture and Mapping Contract

`docs/architecture/script-differences.md` becomes the migration contract document.

Each source script row must include:
- `判定` (`需要覆盖`/`可不覆盖`)
- `处理策略` (`COPY_FROM_MAIN`/`MIGRATE_TO_GO`)
- `目标位置` (target Go file path or copied shell path)
- `目标符号` (method/function or approved unified test suite mapping)
- `迁移状态` (`TODO`/`IN_PROGRESS`/`DONE`/`N/A`)
- `备注` (compatibility, aggregation rationale, risks)

## 5. Execution Phases

### P0.0 Sync Baseline to Upstream v8.31.1

- Fetch upstream tags and refs.
- Freeze baseline at upstream `v8.31.1`.
- Record the exact commit/tag and snapshot date in docs.

### P0.1 Initialize Full Mapping Matrix

- Rebuild full `.sh` inventory from frozen upstream baseline.
- Re-apply classification rules (`需要覆盖`/`可不覆盖`).
- Populate target mapping draft for every file.

### P1 Copy Non-Migrated Scripts

- For all `COPY_FROM_MAIN` entries:
  - record source path (`main@v8.31.1`)
  - record destination path (`go`)
  - copy with executable-bit/shebang checks
  - mark row status accordingly

### P2 Migrate Logic Scripts by Domain

- For all `MIGRATE_TO_GO` entries:
  - inspect original shell behavior
  - map to existing Go packages where possible
  - add/extend packages only where required
  - update row with concrete file+method ownership

Domain landing principles:
- `state/context/validation` -> `internal/tracks`, `internal/context`, `internal/validation`
- `hook/route/provider/intelligence` -> `internal/hooks`, `internal/providers`, `internal/modelroute`
- `workflow/session/task` -> `internal/workflows`, `internal/tracks`, `internal/app`
- `metrics/scheduler/permissions/agent-teams/extract` -> domain packages under `internal/*`
- test shell scripts -> `internal/*_test.go` (same-name unified mapping only where rule allows)

### P3 Verify Contracts and Compatibility

- Verify atomic command contract alignment for migrated capabilities.
- Verify no-shell runtime requirements remain satisfied.
- Verify compatibility facades are documented where retained.
- Close matrix statuses and produce gap list.

### P4 Finalize Docs

- Finalize `docs/architecture/script-differences.md` as source-of-truth matrix.
- Include summary counts by strategy and status.

## 6. Data Flow

1. Source script (upstream `v8.31.1`) is classified.
2. Strategy selected: copy or migrate.
3. Mapping row updated with target file/method.
4. Implementation or copy task executes.
5. Verification updates migration status.
6. Matrix remains the audit trail for completion and parity.

## 7. Error Handling

- Missing upstream tag/ref: block P0.0 and record remediation.
- Ambiguous mapping for non-test script: keep `TODO` and block execution for that row.
- Conflict between shell behavior and current Go architecture: log in `备注`, define compatibility facade, and continue.

## 8. Verification Strategy

- Inventory verification: row count must equal upstream baseline script count.
- Mapping verification: every row has strategy and destination.
- Policy verification: non-test scripts cannot use broad `internal/*_test.go` fallback.
- Traceability verification: all `MIGRATE_TO_GO` rows resolve to file+method-level targets.

## 9. Acceptance Criteria

- `script-differences.md` covers all upstream `v8.31.1` shell scripts.
- Every row explicitly answers copy vs migrate and destination ownership.
- Same-name test uniform mapping rule is applied only where allowed.
- Non-test scripts have domain + per-file concrete mapping.
- Matrix can be directly executed as an implementation backlog.

## 10. Risks and Mitigations

- Upstream drift after baseline freeze.
  - Mitigation: freeze at `v8.31.1`; subsequent changes handled as incremental patch rounds.
- Over-aggregated mappings hide missing behavior.
  - Mitigation: require per-file row ownership and explicit `目标符号`.
- Large migration surface causes status drift.
  - Mitigation: status-managed phased execution and verification gates.
